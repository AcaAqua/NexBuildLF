# 現場向け・単体工程表アプリ 開発方針

## 1. 基本方針

NexBuildLF は、案件管理アプリではなく、現場向けの単体工程表アプリとして育てる。

- 案件管理ではなく工程表中心にする
- 現場監督・小規模工務店・リフォーム会社向けにする
- 低価格・軽量・単体運用を前提にする
- タブレットで使いやすいUIを重視する
- オフラインでも使えることを重視する

## 2. 優先機能

優先して育てる機能は以下とする。

- 工程表
- ガントチャート
- 工程ステータス管理
- 工程削除
- 工程複製
- 工程並び替え
- JSONバックアップ
- Excel出力
- 日報メモ
- 写真メモ
- 注釈付き写真
- 変更履歴タイムライン

## 3. 工程表中心UI

UIは工程表を主役にする。

- ガントを主役にする
- 工程バーをタップして詳細パネルを開く
- 工程リストはガント下部に常時表示する
- 工程ごとに編集・複製・削除・保留を操作できるようにする
- 日付セルや工程バーに記録件数バッジを出す
- ライトテーマ最適化は将来候補とし、現段階では現場での視認性を優先する

## 4. 現場記録機能

工程表のバー・日付に現場記録を紐づける。

- 日報メモ
- 写真
- 手書き注釈付き写真
- 変更理由
- 申し送り
- 資料添付

## 5. 将来のGoogleカレンダー連携

Googleカレンダー連携は将来機能として扱う。

- Googleカレンダーを工程表として再表示する
- 工程をGoogleカレンダー予定へ同期する
- 専用カレンダーを作成する
- 工程カテゴリを設定する
- 通知設定を行う
- 専用タグを追加する
- 買い切りPro版候補として扱う

## 6. 価格構想

価格は買い切り型を基本案とする。

- 単体工程管理アプリ: 買い切り980円案
- Googleカレンダー連携版: 買い切り1,500〜1,980円案
- 同一メジャーバージョン内は軽微修正として扱う
- 大型アップグレード版は別商品扱いにする

## 7. 実装しないこと

現段階では以下は実装しない。

- SQLite
- Googleログイン
- GoogleカレンダーAPI
- クラウド同期
- 決済
- 外部公開
- AI解析
