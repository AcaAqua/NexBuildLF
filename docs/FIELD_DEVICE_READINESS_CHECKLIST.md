# Field Device Readiness Checklist

このチェックリストは、配布そのものではなく、スマホ・タブレットで現場利用できる品質に到達しているかを確認するための受入基準です。

## 対象端末

- Android Chrome: 360px幅のスマートフォン縦向き
- iPhone Safari相当: 390px幅のスマートフォン縦向き
- iPad / Android tablet: 768px幅以上のタブレット縦向き
- Tablet landscape: 1024px幅以上の横向き

## 必須確認

| 区分 | 確認項目 | 合格条件 |
| --- | --- | --- |
| 起動 | ホーム画面追加後の起動 | アプリ名・アイコンで起動し、初回表示が崩れない |
| オフライン | 圏外または通信遮断時 | キャッシュ済み画面が表示され、オフライン表示で状態が分かる |
| 工程表 | ダブルタップ編集 | 工程をダブルタップして編集画面が開く |
| 工程表 | ピンチ拡大縮小 | 指で広げる/つまむ操作で工程表の表示倍率が変わる |
| 工程表 | 横向きタブレット | 工程名、期間、状態、日付が隠れず確認できる |
| 工程編集 | 写真添付 | カメラまたはファイル選択で複数写真を紐づけ保存できる |
| 工程編集 | 大きいタッチ領域 | 主要ボタンが44px以上で誤タップしにくい |
| 日報 | スマホ入力 | 片手操作でも入力欄・保存ボタンが見切れない |
| データ保全 | バックアップ | JSONバックアップに現場・協力業者・設定が含まれる |
| データ保全 | 復元 | 復元前に件数プレビューと上書き警告が表示される |

## 回帰テスト手順

1. `npm run typecheck`
2. `npm run lint`
3. `npm run build`
4. スマホ幅で `/project` を開き、工程の追加、写真添付、保存を確認する。
5. スマホ幅で工程表のダブルタップ編集とピンチ拡大縮小を確認する。
6. タブレット横向きで工程表を開き、工程名・期間・状態・日付の視認性を確認する。
7. `/meeting` で日報入力とタイムライン表示を確認する。
8. `/settings` でバックアップ出力、無効JSONの拒否、有効JSONの復元確認を確認する。
9. PWAとしてホーム画面追加後に起動し、オフライン状態でキャッシュ済み画面を確認する。

## 不合格時の扱い

- 文字の重なり、ボタンの見切れ、横スクロール不能、操作不能は配布可能品質に未達とする。
- 端末依存で再現する場合は、端末名、画面幅、向き、操作手順を記録して修正対象にする。
- データ消失や復元失敗は、UI崩れより優先して修正する。
